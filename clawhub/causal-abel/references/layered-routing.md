# Layered Routing

Use this file when `causal-abel` needs proxy routing for a broad real-world question and a single mega-cap anchor would flatten the story.

## Core Idea

Proxy routing is stronger when anchors represent different economic roles rather than several versions of the same role.

Prefer layers such as:

- supply
- manufacturing
- infrastructure
- platform
- demand
- financing
- policy or macro stress

## Layered Anchor Rules

- Pick 3-5 anchors across distinct layers.
- Include at least one anchor from the opposite side of the decision when that contrast is part of the question.
- Prefer specific mid-cap or supply-chain names over using only the largest platform names.
- Treat anchors as routing hypotheses first, not proof that every node exists on the current CAP surface.

## Suggested Layer Patterns

### AI And Compute

Useful layers:

- design
- manufacturing
- infrastructure
- platform
- China or policy-exposed stack

Example anchors:

- design: `NVDA`, `AMD`, `AVGO`
- manufacturing: `TSM`, `ASML`, `LRCX`
- infrastructure: `DELL`, `ANET`, `VRT`
- platform: `MSFT`, `GOOG`, `AMZN`
- China or policy-exposed stack: `BABA`, `BIDU`, `SMIC`, `USDCNY`

### Consumer Choice

Useful layers:

- convenience platform
- grocery or household budget proxy
- premium or discretionary proxy
- financing or credit stress proxy

Example anchors:

- convenience: `DASH`
- grocery or value retail: `KR`, `WMT`, `COST`
- discretionary: `XLY`
- credit stress: `HYG`

### Education Or Career

Useful layers:

- education provider or signaling layer
- labor market layer
- platform displacement layer
- financing or macro stress layer

Example anchors:

- education: `DUOL`, `LOPE`, `CHGG`
- labor: `UPWK`, `FVRR`
- displacement: `NVDA`, `MSFT`, `ADBE`
- financing or stress: `HYG`

## Convergence Read

Once anchors are chosen:

1. run an independent structural read on each anchor
2. note repeated nodes, repeated bridge types, and repeated proxy dimensions
3. validate the strongest repeated pattern with a narrower structural check
4. only then decide whether mechanism search is needed

## Divergence Read

Different layers do not need to agree.

Sometimes the insight is that:

- supply-side anchors point one way while demand-side anchors point another way
- financing stress contradicts product optimism
- platform winners and labor proxies diverge

When that happens, report the divergence directly instead of forcing a single directional story.

## Guardrails

- do not use layered routing as an excuse to run too many anchors
- do not keep adding anchors once the repeated pattern has stabilized
- do not narrate a layer as causal fact unless the graph or evidence actually supports it
- keep user-facing language on economic roles first; keep raw tickers for trace and verification
