# Route Index

This file is kept as a compatibility entrypoint. It is no longer the main workflow file.

Use the shorter read order:

1. Read `../SKILL.md` and classify the question.
2. Read `routes/capability-discovery.md` as the live-surface module.
3. Read exactly one usage route file:
   - `routes/direct-graph.md`
   - `routes/proxy-routed.md`
4. Read `orchestration-loop.md`.
5. Read `grounding-and-labeling.md` when node mapping or human-facing labels are needed.
6. Read `web-grounding.md` when the current unknown is better answered by external evidence.
7. Read `probe-usage.md` only for exact command shapes.

Do not treat this file as a second copy of the full workflow.
